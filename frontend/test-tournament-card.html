TournamentCard Button Logic Test

Test Scenarios:

  • Joined Room: Button should show "View Room" with success variant and call onView()
  • Non-joined Room (can join): Button should show "Join Room" with primary variant and call onJoin()
  • Full Room: Button should show "Room Full" with secondary variant and be disabled
  • Non-waiting Room: Button should show "View Room" with secondary variant and be disabled

Implementation Details:

// TournamentCard Button Logic:
variant={isJoined ? 'success' : canJoin ? 'primary' : 'secondary'}
disabled={!canJoin && !isJoined}
onClick={() => {
  if (isJoined) {
    onView?.(room.id)  // Direct navigation to room
  } else if (canJoin) {
    onJoin?.(room.id)  // Show confirmation modal
  }
}}

// Button Text:
{isJoined ? 'View Room' : isFull ? 'Room Full' : isWaiting ? 'Join Room' : 'View Room'}
